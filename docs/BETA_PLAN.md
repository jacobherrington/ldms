# LDMS v0.1.0 Beta Plan (2 weeks)

## Objective

Validate that new users can reach first successful usage in under 5 minutes using container-first setup.

## Cohort

- 10-20 external engineers across mixed environments
- At least 3 users with no Ruby toolchain preinstalled

## Success metrics

- 80% complete smoke + first memory save
- Median time-to-first-success under 5 minutes
- Top 3 onboarding blockers identified and prioritized

## Timeline

### Week 1
- Recruit beta users and share quick-start docs.
- Collect setup friction from issues and short survey.
- Triage daily and patch critical onboarding blockers.

### Week 2
- Validate fixes with fresh users.
- Confirm CI and docs are stable for v0.1.0.
- Finalize changelog and release notes.

## Feedback intake

- Bugs: GitHub bug template
- Feature gaps: GitHub feature template
- Security concerns: private report via `SECURITY.md`
